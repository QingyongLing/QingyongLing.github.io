记一次笔试编程题

发表于 2017-03-21 | 分类于 LeetCode

今儿个写了道笔试编程题，不难，但是还是写错了，Orz，还是思维不够缜密啊，在此记录一下，同时表示深感智商捉急。
题目很简单，见LeetCode。
这题目要求的是一个二叉树的两个节点的最近公共祖先，但是我做的笔试题要求不能使用额外的空间存储保存树的结构，也就是要求O（1）的空间复杂度。
这道题在了LeetCode做过，用的方法很直接，可以说简单粗暴，直接保存遍历的路径两条路径中相等的节点的最后一个节点。

这里用两个vector<TreeNode*>分别保存要查找的两个节点的遍历路径，然后找到两个路径中的最后个相同的节点，此节点就可以把最近的公共祖先找到。
因此这个方法是O(n)的时间复杂度和O(n)的空间复杂。
然而第二次做这个笔试题的时候就明说了不能保存节点的信息，因此没有想清楚我就写了个方案。

没有仔细思考就给了这个方案，现在觉得实在是智商捉急，也是没有经过仔细的测试验证导致的。这里面的一个致命错误是，当一个节点被找到了，然后找到的第二个节点会马上被当成祖先节点，然而实际上这个节点并不是（ORZ）。
因此这个方案是完全错误的。来分析一下具体的情况：
(1)两个节点相同
(2)两个节点中一个节点是另一个节点的祖先
(3)两个节点一个在左子树一个在右子树（相对于第一个公共祖先而言）

(1)两个节点相同的时候可以返回当前的节点
