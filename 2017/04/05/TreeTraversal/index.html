遍历二叉树

发表于 2017-04-05 | 分类于 数据结构

二叉树的遍历是一个很简单的东西，递归的写法三行两就写完了，非递归的写法以前做题的时候是模拟栈特性来做，今天看到个Morris遍历，感觉比较有趣，只需要O(N)的时间复杂度和O(1)的空间复杂度。
首先遍历分为前序、中序和后序三种方式，这三种方式的区别在于遍历的次序。前序遍历先遍历自身，然后遍历左子树，最后遍历右子树。中序遍历先遍历左子树，然后遍历自身，最后遍历右子树。后序遍历先遍历左子树，再遍历右子树，最后遍历自身。

二叉树的递归遍历

这个就很简单了，只要按照三种遍历的次序不同，简单的更改代码执行的顺序就好了。

非常简单，不再解释。不过说一下的是在递归过程中因为需要运行时栈内存，因此对于非平衡树空间复杂度为O(N)，对于平衡树而言递归的空间复杂度为O(logN)，时间复杂度均为O(N)。

二叉树的非递归遍历

在以前写题的时候写过非递归遍历分别对应以下的LeetCode题号：
LeetCode 144. Binary Tree Preorder Traversal
LeetCode 94. Binary Tree Inorder Traversal
LeetCode 145. Binary Tree Postorder Traversal
第一个是前序遍历，前序遍历要求先输出当前的节点，然后再遍历左子树，最后遍历右子树。那么如何在循环中模拟递归的问题，我们用一个栈（或者vector也可以）模拟递归的过程，把栈顶的元素作为当前节点。因此步骤是：
（1）把树的根节点放入栈
（2）把栈顶元素作为当前节点，并取出（pop），输出该节点
（3）如果当前节点右子树存在，把右子节点入栈
（4）如果当前节点左子树存在，把左子节点入栈
（5）重复2，3，4步，直到栈为空
这个过程和递归的思想基本一致，保存右子节点，然后再保存左子节点，同时去除当前节点（已经遍历）。右子节点先入栈，后出栈，左子节点后入栈，先出栈，因此右子节点（右子树）比左子节点（左子树）晚遍历，下一次循环的时候，相当于当前节点变为上一次的左子节点（如果存在），然后按照上面的模式把左子树都遍历了一遍，全部遍历完了，再遍历右子树，保证了遍历的顺序。
对于一次循环而言，无法知道当前节点处于何处，因此只需要把右子树放在左子树的后面遍历就可以，只要每个节点都保证这个顺序，那么整个先序遍历就是正确的。此处也利用了栈的特性，在栈顶的优先被遍历。这又有点递归的感觉了，不过不是用递归来写，也有点像数学的归纳法（每个节点满足顺序，整个遍历就是对的）。

接下来写中序遍历，中序遍历要求先遍历左子树，然后输出当前节点，最后遍历右子树。那么那么用栈怎么实现呢，思考递归的方式，当我们遍历一棵树的时候，传进来一个根节点，代码会一直递归直到整棵树最左边的节点，然后输出该节点，再遍历该节点的右子树。然后返回到上一层递归。
因此最重要的是如何用栈遍历左子树，然后输出当前节点，再遍历右子树。这里直接模拟递归调用，对于一个root节点，把它的左子节点一直放入栈里面（相当于递归遍历左子树的过程），当到了最左边的时候，把栈顶节点遍历（输出当前节点），然后出栈，同时把该节点的右子节点当成新的根节点重复这一过程。
一个元素出栈的时候，它的左子节点已经出栈（输出了），这也就意味着左子树已经遍历完了，它的右子节点会该节点出栈后马上入栈，接着遍历右子树。整个过程能否满足左子树都在当前节点的前面，右子树都在后面？注意我们的循环过程，每一次都把root当成一个新的根节点，然后一路往左，直到最左边，然后出栈，遍历该节点，再把右子树当成一棵新的树来处理。因此我们可以确保每个节点都被遍历，同时保证遍历顺序。

后序遍历要复杂一些，后序要求遍历左子树在，然后遍历右子树，最后输出自身节点，关键的问题在于如何确定当前这个节点要被输出还是先遍历其右子树。这里分两种情况，第一种是右子节点不存在，第二种是右子节点存在。那么对于一个当前节点来说就要根据这两种情况作出区分。
我们用一个指针记录上一次被遍历的节点。因为在后序遍历中，根节点是最后被遍历的，在遍历子树的时候最后遍历的必然是左子节点或者是右子节点，根据这个特性可以区分出左右子树是否已经被遍历过了。可以给出以下区分的方法：
（1）当前右子节点不存在，那么应该输出当前节点，并且更新last指针
（2）当前的子节点存在，而且last指针不指向右子节点，那么应该遍历右子树，因此把右子树作为新的树遍历
（3）当前的右子节点存在，last指针指向右子节点，因此左右子树都已经遍历完了，可以遍历该节点并且更新last

在这里中序和后序的解决方法有些相似之处，就是说对于一棵树而言，我们模拟递归的过程，只要拿到了一棵树就马上一路往左，直到最左边的节点，然后再以处理一颗新的树的形式来处理每个节点的右子树。每次弹出一个节点都代表该节点及其子树都被遍历。

二叉树的Morris遍历

Morris遍历用到了线索二叉树的概念，利用空闲的叶子节点的指针来找后继结点，最后实现O(n)时间复杂度和O(1)的空间复杂度的遍历。
这里先写中序遍历，步骤为：
（1）如果当前节点的左子节点为空，则输出当前的节点，并且把它的右子节点作为当前节点
（2）如果当前节点的左子节点不为空，那么寻找该节点在左子树中的前驱节点（即左子树的最右节点）
（2.a）如果前驱节点的右子节点为空，那么将它的右子节点设置为当前节点，并且把当前节点的左子节点作为新的当前节点
（2.b）如果前驱节点的右子节点为当前节点，那么将其右子节点重新设置为空（恢复树形状），输出当前节点，把当前节点的右子节点作为新的当前节点
（3）重复上述1，2的步骤直到当前节点为空
非常巧妙的是在左子树遍历之前，把根节点的前驱节点的右儿子指向跟节点，然后遍历左子树，最后通过该前驱节点回到原来的根节点，并且把修改过的左子树复原。
这里如何保持中序，首先如果左子树不存在，那么直接输出好了，然后转向右子树，当左子树存在的时候，当前节点只有在发现前驱节点指向自己时输出，那么左子树遍历完了，可以输出当前节点。
实际上，每一次要返回上一层的时候，都是通过提前设置的前驱节点的右儿子来实现的。

代码逻辑还是比较清楚地，步骤第一个如果左子节点不存在，输出，转向右子树；左子树存在的时候去找到前驱，然后判断前驱是否指向自身，如果指向自身那么输出，如果没有指向自身那么就把前驱节点指向当前节点。如果要比较好的理解这个算法，还是要一步一步画个图比较好，手动画图对照程序可以有更深入的理解。
接下来看前序遍历的步骤：
（1）如果当前节点的左子节点为空，则输出当前的节点，并且把它的右子节点作为当前节点
（2）如果当前节点的左子节点不为空，那么寻找该节点在左子树中的前驱节点（即左子树的最右节点）
（2.a）如果前驱节点的右子节点为空，那么将它的右子节点设置为当前节点，输出当前节点，并且把当前节点的左子节点作为新的当前节点
（2.b）如果前驱节点的右子节点为当前节点，那么将其右子节点重新设置为空（恢复树形状），把当前节点的右子节点作为新的当前节点
（3）重复上述1，2的步骤直到当前节点为空
跟中序不同的地方只有一点，那就是输出的地方不一样。

Morris后序遍历比较麻烦，要求建立一个临时节点，并且其将root作为其的左子节点，同时需要两个节点之间路径翻转两次。步骤为：
（1）设立临时节点dump，并且将root作为其左子节点，然后从dump遍历
（2）如果当前节点的左子节点不存在，那么将其右子节点作为当前节点
（3）如果当前节点的左子节点不为空，那么寻找该节点在左子树中的前驱节点（即左子树的最右节点）
（3.a）如果前驱节点的右子节点为空，那么将它的右子节点设置为当前节点，并且把当前节点的左子节点作为新的当前节点
（3.b）如果前驱节点的右子节点为当前节点，那么将其右子节点重新设置为空（恢复树形状），倒序输出从当前节点的左孩子到前驱节点路径上的所有节点，把当前节点的右子节点作为新的当前节点
（4）重复上述2，3的步骤直到当前节点为空
确实略麻烦,在输出的时候要求单向链表的倒序输出，因此需要注意的改变链表方向，并且在完成翻转以后还要翻转回来。

后序遍历每次遍历都要更改树的结构，然后再复原，但是其时间复杂度依旧是O(N)的。
