Tree Restoration

发表于 2017-04-21 | 分类于 Algorithm

这也是一道编程题，当时写了好一会，这道题不难，题目的逻辑也很清楚，关键在于复现人在处理这个问题的思维。

Tree Restoration

There is a tree of N nodes which are numbered from 1 to N. Unfortunately, its edges are missing so we don’t know how the nodes are connected. Instead we know:

  1. Which nodes are leaves
  2. The distance (number of edges) between any pair of leaves
  3. The depth of every node (the root’s depth is 1)
  4. For the nodes on the same level, their order from left to right
    Can you restore the tree’s edges with these information? Note if node u is on the left of node v, u’s parent should not be on the right of v’s parent

输入

The first line contains three integers N, M and K. N is the number of nodes. M is the depth of the tree. K is the number of leaves.
The second line contains M integers A1, A2, … AM. Ai represents the number of nodes of depth i.
Then M lines follow. The ith of the M lines contains Ai numbers which are the nodes of depth i from left to right.
The (M+3)-th line contains K numbers L1, L2, … LK, indicating the leaves.
Then a K × K matrix D follows. Dij represents the distance between Li and Lj.
1 ≤ N ≤ 100

输出

For every node from 1 to N output its parent. Output 0 for the root’s parent.

样例输入

8 3 5
1 3 4
1
2 3 4
5 6 7 8
3 5 6 7 8
0 3 3 3 3
3 0 2 4 4
3 2 0 4 4
3 4 4 0 2
3 4 4 2 0

样例输出

0 1 1 1 2 2 4 4

解析

首先这道题给出了总节点个数（N），树的深度（M），叶子节点个数（K）。
接下来是每一层的叶子节点的个数。
接下来M行分别说明了每一层对应那些节点。
接下来就是K x K的矩阵，代表每个叶子节点之间的距离。
要求求解每个节点父节点，根节点的父节点为0。
这道题实际上是要我们根据叶子节点，以及叶子节点的距离重建这颗多叉树。考虑这样的一个事实，如果两个叶子节点有相同的父节点，那么它们之间的距离应该为2。
我们有每一层的节点信息，因此对于最底层的节点，必然都是叶子节点，此时它们分别对应其父节点，相当于一个多对多的关系，但是它们之间的对应关系可以根据叶子节点的距离来界定。
如图中，5和6的距离是2，因此是拥有一相同父节点，6和7之间距离不是2，因此拥有不同的父节点，7和8拥有相同的父节点，此时看上一层的信息，有2和4是非叶子节点，所以叶子节点被划分为两组，分别对应父节点2和4。
处理完了最底下的那一层，接下来如何处理上面的节点？我们要做的是把倒数第二层也当成叶子节点来处理，并且按这种思路逐步向上，直到根节点。
这是需要处理的关键问题就在于维护新的叶子节点的距离信息，新的叶子节点是原来叶子节点的父节点，所以该节点与其他节点的距离可以通过旧的叶子节点求出来。

整个代码比较长，首先nodes保存的是每一层的节点信息，leafs维护的是叶子节点的集合，leafvec保存的初始时刻的叶子节点，fathers用于记录每个节点的父节点，distan记录叶子节点之间的距离。
代码主要过程是：
(1)输入数据，把各相关数据初始化，记录初始时刻的叶子节点集合和每个叶子节点距离。
(2)找到上一层的首个非叶子节点，下标对应为fatherindex（对应下一层的最靠前的叶子节点）
(3)把下一层的第一个叶子节点的父节点设置为fatherindex对应节点，更新其他叶子节点到父节点的距离，从叶子节点中删除当前叶子节点
(4)寻找下一个同一个父节点的叶子节点，保存其父节点，并从leafs集合中删除
(5)把上面叶子节点的父节点加入到叶子节点的集合
(6)进入(2)，重复这一过程，直到到达树的根节点
其实只要想清楚了处理过程还是很容易理解的。
